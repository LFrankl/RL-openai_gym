CartPole-v1

稠密奖励 入门

小车上竖一根杆，左右施力保持平衡，每撑过一步得 +1，最高 500 分。

算法核心思路评估均分
Q-Learning 状态离散化 + Q-Table ~351
DQN 神经网络 + 经验回放 + Target Net ~190
PPO Actor-Critic + GAE + Clip ~212
关键发现：简单任务里，传统 Q-Table 反而比神经网络更稳。 状态空间低维时，离散化不是瓶颈，网络的过拟合才是。
⛰️

MountainCar-v0

极稀疏奖励 探索难题

山谷小车，动力不足以直接冲顶，必须左右摇摆积累势能。每步 -1， 随机策略在 10,000 回合内到顶次数：0 次。

算法方法到顶率
Q-Learning（原始） ε-greedy 失败
Q-Learning（塑形） 奖励塑形 + 乐观初始化 失败
DQN（多种奖励） 差分势函数 / Bootstrap 失败
PPO（多种奖励） 速度势 / 高度势 失败
行为克隆（BC） 专家演示 + 监督学习 100%  -118分
核心结论：奖励极稀疏时，RL 探索机制根本触碰不到正反馈。 行为克隆绕过探索问题，直接从专家演示中学习——有时模仿比探索有效得多。
🚀

LunarLander-v3

稠密奖励 多目标

飞船从高空下落，稳稳降落在着陆垫。≥ 200 分视为解决，三代 DQN 网络参数量完全相同。

算法改进点均分解决
DQN 基准 185.9 ✗
Double DQN 解耦动作选择与价值评估 155.0 ✗
Dueling DQN V(s)+A(s,a) 分解 208.6 ✓
算法演进：三代 DQN 参数量相同，差异纯粹来自架构设计。 Dueling 通过将 Q 值分解为状态价值 $V(s)$ 和优势函数 $A(s,a)$， 让网络对与动作无关的状态信息学习更高效。
🏓

ALE/Pong-v5

图像输入 Atari

状态不再是低维向量，而是 $(210, 160, 3)$ RGB 图像，需要 CNN 提取时序特征。

技术说明
帧堆叠 × 4 恢复 Markov 性质（单帧无法感知运动方向）
图像预处理管道 灰度化 → 裁剪 → 缩放 84×84 → 归一化
CNN 主干 Conv×3 → FC(512) → Q ∈ ℝ⁶，参数量 1.7M
Double DQN 目标 消除图像特征空间下的 Q 值高估
核心洞察：单帧图像违反 Markov 性质——同一画面可能对应 "球向左"或"球向右"两种完全不同的状态，最优动作相反。 帧堆叠通过像素差异隐式编码运动信息，恢复 Markov 性质。
理论基础

核心数学推导

每个算法背后的严格理论——Bellman 方程、策略梯度定理、GAE 推导。

Bellman 最优方程 Q-Learning / DQN

$$Q^*(s, a) = \mathbb{E}_{s'}\!\left[R(s,a) + \gamma \max_{a'} Q^*(s', a')\right]$$

这是一个不动点方程。Bellman 算子 $\mathcal{T}$ 是关于 $\sup$-范数的 $\gamma$-收缩映射，由 Banach 不动点定理保证唯一解。 Q-Learning 以随机近似的方式逐步逼近该不动点。

TD 误差与收敛性 Q-Learning

$$\delta_t = r_t + \gamma \max_{a'} Q(s_{t+1}, a') - Q(s_t, a_t)$$

Watkins & Dayan (1992) 证明：在所有状态-动作对被无限次访问， 且学习率满足 Robbins-Monro 条件（$\sum \alpha_t = \infty$，$\sum \alpha_t^2 < \infty$） 时，Q-Learning 以概率 1 收敛到 $Q^*$。

广义优势估计 GAE PPO

$$\hat{A}_t^{\text{GAE}(\gamma,\lambda)} = \sum_{l=0}^{\infty}(\gamma\lambda)^l \delta_{t+l}$$

以 $\lambda$ 在单步 TD（高偏差低方差）与蒙特卡洛（低偏差高方差）之间做插值。 $\lambda=0$ 退化为 TD(0)，$\lambda=1$ 退化为蒙特卡洛， 有效折扣因子变为 $\gamma\lambda$。

PPO Clip 目标 PPO

$$L^{\text{CLIP}} = \mathbb{E}_t\!\left[\min\!\left(r_t A_t,\ \text{clip}(r_t, 1{\pm}\epsilon) A_t\right)\right]$$

其中 $r_t = \pi_\theta / \pi_{\theta_{\text{old}}}$ 为概率比。 取 min 保证目标是悲观下界（pessimistic bound）， Clip 操作近似 TRPO 的 KL 约束，用一阶优化代替二阶 Fisher 矩阵。

Dueling 网络分解 Dueling DQN

$$Q(s,a;\theta,\alpha,\beta) = V(s;\theta,\beta) + \left(A(s,a;\theta,\alpha) - \frac{1}{|\mathcal{A}|}\sum_{a'}A(s,a')\right)$$

将 Q 值分解为状态价值 $V(s)$ 和优势函数 $A(s,a)$。 减去均值消除不可识别性，使 $V(s)$ 和 $A(s,a)$ 的学习信号相互独立， 在无需改变动作的状态上更高效地更新 $V$。

帧堆叠与 Markov 性质 CNN DQN

$$s_t = (f_{t-3}, f_{t-2}, f_{t-1}, f_t) \in [0,1]^{4 \times 84 \times 84}$$

单帧图像违反 MDP 的 Markov 性质——无法从静态像素感知球的运动方向。 将最近 $k=4$ 帧拼接后，连续帧的像素差异隐含了速度信息， 从而将 POMDP 近似恢复为 MDP。

MountainCar 探索实录

八次尝试，一次成功

MountainCar 是这个项目里最艰难的游戏。完整记录了所有失败路径—— 它们和成功一样重要。

✗

尝试 1：Q-Learning（原始 ε-greedy）

随机探索在 10,000 回合内几乎不可能自然到顶。Q-Table 对未访问区域的估计全为 0，无法引导探索。均分 -200。

✗

尝试 2：Q-Learning + 乐观初始化

将 Q-Table 初始化为正值，理论上鼓励探索未知区域。但状态空间离散化粒度不足，高价值区域仍触达不到。

✗

尝试 3：Q-Learning + 奖励塑形（速度势函数）

用势函数 $F(s,s') = \gamma\Phi(s') - \Phi(s)$ 添加辅助奖励，理论上不改变最优策略。 但势函数设计错误时会引导策略原地振荡而不到顶。

✗

尝试 4–5：DQN + 差分势函数 / Bootstrap

引入神经网络，尝试多种奖励塑形方案。函数近似反而放大了 Bootstrap 误差，Q 值估计发散，均分 -200。

✗

尝试 6–7：PPO + 速度势 / 自定义奖励

Actor-Critic 对奖励信号设计极其敏感。自定义奖励虽然帮助策略"学会摇摆"，但无法产生真正到达山顶所需的完整轨迹。

✓

尝试 8：行为克隆（BC）——100% 到顶

彻底绕过探索问题。用最优策略 $a^*(s) = \text{sign}(\dot{x})$ 生成专家演示， 监督学习拟合策略。训练后评估均分 -118， 到顶率 100%。核心洞察：有时候问题的难点在于探索本身，而不是学习能力。

快速上手

在本地运行实验

项目使用 uv 管理依赖，Python 3.11，支持 CPU 和 Mac M4 MPS 加速。

1

克隆仓库

git clone https://github.com/LFrankl/RL-openai_gym.git
cd RL-openai_gym
2

安装依赖

需要 uv 包管理器。

# 安装 uv（如果没有）
curl -LsSf https://astral.sh/uv/install.sh | sh

# 同步依赖
uv sync
3

运行任意算法

# CartPole DQN
uv run python cartpole/dqn.py

# LunarLander Dueling DQN（已解决）
uv run python lunarlander/dueling_dqn.py

# MountainCar 行为克隆（100% 到顶）
uv run python mountaincar/dqn_shaped.py

# Pong CNN DQN（Atari 图像输入）
uv run python pong/dqn_cnn.py
4

Mac M4 GPU 加速

所有文件都预留了 MPS 注释，取消注释即可启用 GPU。

# 找到文件中的这一行：
DEVICE = torch.device("cpu")
# 取消注释下一行：
# DEVICE = torch.device("mps")
